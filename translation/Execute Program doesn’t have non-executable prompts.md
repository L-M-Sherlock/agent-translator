# Execute Program 没有不可执行的卡片

[Execute Program doesn’t have non-executable prompts](https://notes.andymatuschak.org/zYCRKm7HujiD6CNtcC9gnmk)

如[读者通过执行一个程序来回答 Execute Program 的卡片](https://notes.andymatuschak.org/z9DRugkwcBv7iLRACEnWVDL)所述，[Execute Program](https://notes.andymatuschak.org/zSsjk5UvNPGrYp8B6DEFCMS) 没有其他类型的卡片。

例如，一节关于正则表达式的课程可能有这样一张卡片：

> Q. `/a..b/.test(“a\ncb”)` 的输出是什么？
> A. `false`

读者会通过在 REPL 中实际输入 `false`，并执行该表达式来回答这张卡片。

但这节课不能同时拥有这种更典型的[间隔重复记忆系统](https://notes.andymatuschak.org/z2D1qPwddPktBjpNuwYFVva)卡片：

> Q. 在许多实现中，`.` **不**匹配哪些字符？
> A. `\n`

主要是概念性的课程（例如 [SQL Constraint Analysis](https://www.executeprogram.com/courses/sql/lessons/constraint-analysis)）不太适合这种模式。

因为 Execute Program 只有一种卡片类型：[Execute Program 的卡片既是应用卡片，也是回忆卡片](https://notes.andymatuschak.org/zUoZHzEPeQFKaUTYctg9fqq)。

------

Q. [Execute Program](https://notes.andymatuschak.org/Execute_Program) 无法编码的一张卡片，有什么例子？
A. [例如：「Q. `*` 和 `+` 有什么区别？A. `*` 允许零次匹配」](https://notes.andymatuschak.org/zYCRKm7HujiD6CNtcC9gnmk)

Q. 为什么 [Execute Program](https://notes.andymatuschak.org/Execute_Program) 的 SQL constraint analysis 课程无法使用它的间隔重复系统？
A. 那一课主要是概念性的，其想法很难编码成可执行的表达式。
