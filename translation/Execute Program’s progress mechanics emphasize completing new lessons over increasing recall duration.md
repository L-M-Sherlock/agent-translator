# Execute Program 的进度机制强调完成新课，而不是延长回忆时长

[Execute Program’s progress mechanics emphasize completing new lessons over increasing recall duration](https://notes.andymatuschak.org/zHJBriKPXFC2VtF8MVCdj7K)

[Execute Program](https://notes.andymatuschak.org/zSsjk5UvNPGrYp8B6DEFCMS) 上的课节有四种状态：

1. Locked（当一个或多个先修内容未达到 Learned；见[在你成功复习其先修内容之前，Execute Program 的课程不会解锁](https://notes.andymatuschak.org/zHLWyC5LVXZLWE9RmoC1gua)）
2. Unlocked（当所有先修内容都已达到 Learned，但你尚未学习该课节）
3. Active（当你已学习该课节，但尚未复习）
4. Learned（当你至少复习过该课节一次）

这种展示到「Learned」就封顶了：读者只要复习过一次课节就能达到它。它不会区分「已证明能记住一个月」和「已证明能记住一天」。

最上面的圆圈代表课程中处于 Learned 状态的课节比例。因此，当所有课节都达到 Learned 时，你就「完成」了这门课，即便它们的 SRS 间隔可能只有一两天。

这与 [Quantum Country](https://notes.andymatuschak.org/zNVVrcxnkjk7pCabiBvaic1) 形成了一个有趣的对比：在后者中，进度的核心是达到越来越高的回忆耐久度。

这种框架的一个有趣后果是：当用户把一门课里的所有课节都复习过一遍之后，他们大体上会觉得自己已经「结束」了。他们仍然会收到复习通知，但 UI 并不会把这些复习与任何有意义的目标关联起来，因此很多用户很可能会在这个节点流失。

------

Q. 在 Execute Program 上，「完成」一节课的标准是什么？
A. 学过它，并至少复习过一次。

Q. Execute Program 的进度机制的侧重点与 Quantum Country 有何不同？
A. 它强调先把课程课节学完的初始进度，而不是强调在长尾阶段不断增加 SRS 间隔。
