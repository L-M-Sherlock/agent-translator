# 在 Execute Program 的复习中，错过的答案不会惩罚学生，除非他们「放弃」

[Missed answers in Execute Program’s reviews don’t penalize the student unless they “give up”](https://notes.andymatuschak.org/zX7RdiHGbHYnkFznUr9VZx7)

[Execute Program](https://notes.andymatuschak.org/zSsjk5UvNPGrYp8B6DEFCMS) 的[间隔重复记忆系统](https://notes.andymatuschak.org/z2D1qPwddPktBjpNuwYFVva)卡片是机器判分的，因此如果读者在回答中出现语义上无关紧要的拼写错误，系统也会把卡片判错。这种假阴性会非常恼人，尤其是因为[在你成功复习其前置课程之前，Execute Program 的课程不会解锁](https://notes.andymatuschak.org/zHLWyC5LVXZLWE9RmoC1gua)。在 SQL 课程中，假阴性率高到足以促使系统在 2020 年初进行修改：只要学生最终能给出正确答案，不正确的回答就不再被「算作」失败。读者可以想尝试多少次就尝试多少次，而下一次复习的间隔仍会按 SRS 曲线增加。后续课节的解锁也不再取决于用户表现。

如果用户在尝试多次后仍无法给出答案，他们最终可以点击「give up」来查看正确答案。如果他们以这种方式放弃，那么该问题的间隔会像 SRS 中常见的「遗忘答案」那样缩短。

另见：[自评式间隔重复记忆系统能避免假阴性](https://notes.andymatuschak.org/zQp7to6CDgQzSFqM7k3yPn7)以及[间隔重复记忆系统中的自评与机器判分](https://notes.andymatuschak.org/zScaVuH9r2pwo24iD3f39bU)。
